## Огляд проєкту

Цей репозиторій створено для виконання практичного завдання з автоматизації процесів **Неперервної Інтеграції та Доставки (CI/CD)** з використанням GitHub Actions та хмарних сервісів Microsoft Azure.

**Мета роботи:** створити робочий процес, який виконує збірку (build) front-end проєкту, публікує його як Docker-образ у **GitHub Container Registry (GHCR)** та автоматично розгортає (deploy) у **Azure App Service**.

## Використані технології

* **Система автоматизації:** GitHub Actions
* **Контейнеризація:** Docker
* **Реєстр образів:** GitHub Container Registry (GHCR)
* **Хмарний хостинг:** Azure App Service
* **Середовище збірки:** Node.js
* **Веб-сервер:** Nginx

## GitHub Actions Workflow (`.github/workflows/ci-cd.yml`)

Робочий процес `ci-cd.yml` реалізує повний цикл доставки ПЗ.

### Тригери (Запуск)
Workflow запускається автоматично при пуші в гілки `main` або `feature/*`, а також вручну через `workflow_dispatch`.

### Етапи виконання

#### 1. Build and Push (`build-and-push`)
* Клонування репозиторію.
* Встановлення залежностей та збірка проєкту (`npm run build`).
* Авторизація в GitHub Container Registry.
* Збірка Docker-образу та його публікація в GHCR.

#### 2. Deploy to Azure (`deploy`)
Цей етап виконується після успішного завершення збірки:
* **Azure Login:** Авторизація в Azure за допомогою Service Principal (використовуючи секрет `AZURE_CREDENTIALS`).
* **Deploy to Azure Web App:** Розгортання Docker-образу з GHCR у створений ресурс Azure App Service.

## Висновки

В ході виконання лабораторних робіт, присвячених темі CI/CD (Неперервна Інтеграція та Доставка), я реалізував повний цикл автоматизації розробки програмного забезпечення:

**1. Етап Continuous Integration (CI):**
* Опанував роботу з **GitHub Actions** та налаштував workflow, який автоматично реагує на зміни в коді (push у гілку).
* Навчився автоматизувати збірку Docker-образів та публікувати їх у реєстрі контейнерів **GitHub Container Registry (GHCR)**.
* Забезпечив версійність артефактів та безпеку процесу збірки.

**2. Етап Continuous Delivery (CD):**
* Ознайомився з хмарною платформою **Microsoft Azure** та створив інфраструктуру для хостингу веб-застосунку (Azure App Service).
* Налаштував безпечну взаємодію між GitHub та Azure за допомогою **Service Principal**, що дозволяє виконувати дії без зберігання паролів у відкритому вигляді.
* Додав етап автоматичного розгортання (Deployment), завдяки чому нова версія сайту стає доступною користувачам одразу після успішного проходження тестів та збірки.

**Загальний підсумок:**
У результаті я побудував повноцінний пайплайн (pipeline), який дозволяє пройти шлях від написання коду локально до його появи на "живому" сервері без ручного втручання. Це значно прискорює процес розробки, зменшує ризик людських помилок та забезпечує швидкий зворотний зв'язок.
